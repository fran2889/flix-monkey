# Multi-Platform Service Architecture Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Refactor FlixMonkey to support multiple streaming platforms by introducing a service abstraction layer, separating Netflix-specific code from core application logic

**Architecture:** Service classes define platform metadata and surface managers; ServiceRegistry detects current platform by domain; SurfaceManager becomes platform-agnostic base class with platform-specific subclasses; OverlayRenderer accepts service-specific constants

**Tech Stack:** JavaScript ES2022, existing FlixMonkey codebase, Vitest for testing

## Global Constraints

- All files in `src/` and `tests/` must include GPL-3.0 license header
- Follow existing code style: ES modules, private fields with `#` syntax
- Maintain backward compatibility: Netflix must work identically
- All existing tests must continue to pass
- Conventional Commits format for all commits
