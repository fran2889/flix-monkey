# CSS Fixes: Overlay & Settings UI — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Fix 12 CSS issues across overlay badges and the settings UI: missing rules, unscoped globals, inline styles bypassing the CSS system, and accessibility/forward-compatibility gaps.

**Architecture:** Three files touched: `overlay.js` (scoped globals, font-family, redundant declaration), `styles.js` (missing rules, structural cleanup, accessibility), `settings-ui.js` + `styles.js` together (validation errors → status area, status colour via class). Tests in the matching test files are updated first (TDD) wherever behaviour changes.

**Tech Stack:** Vanilla JS, CSS-in-JS strings, Vitest, jsdom

## Global Constraints

- No new dependencies.
- All CSS changes are inside the existing `SETTINGS_STYLES` string (`src/core/ui/styles.js`) or the `cssText` template literal in `injectStyles()` (`src/core/overlay.js`).
- Colour values must come from the project palette: `#e05252` (error red), `#4caf50` (success green), `#e50914` (Netflix red), `#aaa` (muted).
- Run `npm test` (unit + UI) after each task; all tests must pass before committing.

- [x] **Step 3: Rewrite `#validate()` in `src/core/ui/settings-ui.js`**

Replace the entire `#validate()` method (lines 191–224):

```js
#validate() {
    const errors = [];
    const allValues = {};
    this.#fields.forEach(field => {
        const input = this.#container.querySelector(`#fm-${field.key}`);
        if (!input) return;
        allValues[field.key] = input.type === 'checkbox' ? input.checked : input.value;
    });
    this.#fields.forEach(field => {
        const input = this.#container.querySelector(`#fm-${field.key}`);
        if (!input) return;
        const fieldValue = input.type === 'checkbox' ? input.checked : input.value;
        const errorMsg = field.validate ? field.validate(fieldValue, allValues) : null;
        if (errorMsg) {
            errors.push(errorMsg);
            input.classList.add('error');
        } else {
            input.classList.remove('error');
        }
    });
    return errors;
}
```

- [x] **Step 4: Update `save()` in `src/core/ui/settings-ui.js`**

Replace lines 226–256 (the `save()` method):

```js
async save() {
    const errors = this.#validate();
    const statusDiv = this.#container.querySelector('#fm-status');

    if (errors.length > 0) {
        statusDiv.textContent = errors.join('\n');
        statusDiv.className = 'fm-status--error';
        return;
    }

    const values = {};
    this.#fields.forEach(field => {
        const input = this.#container.querySelector(`#fm-${field.key}`);
        if (field.type === 'checkbox') {
            values[field.key] = input.checked;
        } else {
            values[field.key] = input.value;
        }
    });

    const saveBtn = this.#container.querySelector('#fm-saveBtn');
    if (saveBtn) saveBtn.disabled = true;
    try {
        await this.#adapter.storageSetMany(values);
        statusDiv.textContent = 'Saved!';
        statusDiv.className = 'fm-status--success';
        await this.#onSave?.();
    } finally {
        if (saveBtn) saveBtn.disabled = false;
    }
}
```
